# Модуль Bitrix24 "Группировка потоков" (custom.flowgroups)

## Обзор модуля

Модуль предназначен для организации потоков задач в Bitrix24 в логические группы/разделы. Это позволяет сократить количество строк, отображаемых пользователям, избежать хаоса при большом количестве потоков и сгруппировать потоки по смыслу.

## Функциональность

### Основные возможности:
1. **Создание групп потоков** - администраторы могут создавать группы с названием, описанием и списком ID потоков
2. **Группировка на странице потоков** - автоматическая замена стандартной таблицы потоков на сгруппированное представление
3. **Раскрытие/скрытие групп** - пользователи могут разворачивать/сворачивать группы для просмотра вложенных потоков
4. **Визуальные улучшения** - стилизованные заголовки групп с информацией о владельце, количестве задач, эффективности
5. **Административный интерфейс** - управление группами через раздел администрирования Bitrix24

### Технические особенности:
- Работает только на странице `/tasks/flow/`
- Добавляет CSS и JavaScript для модификации интерфейса
- Использует собственную таблицу `b_custom_flow_groups` для хранения данных
- Интегрируется через обработчик событий `OnBeforeProlog`

## Установка на продакшен

### Предварительные требования
1. Bitrix24 с модулем "Задачи" (tasks)
2. Таблица потоков `b_tasks_flow` должна существовать
3. Права администратора для установки модуля

### Пошаговая установка:

1. **Скопируйте модуль** в директорию:
   ```
   /local/modules/custom.flowgroups/
   ```

2. **Установите модуль** через Marketplace Bitrix24:
   - Перейдите в "Настройки" → "Маркетплейс" → "Установленные решения"
   - Нажмите "Добавить решение" и выберите директорию модуля
   - Или используйте CLI: `php /bitrix/modules/custom.flowgroups/install/index.php`

3. **Проверьте установку**:
   - В списке модулей должен появиться "Группировка потоков"
   - В админке появится пункт "Группировка потоков" в разделе "Сервисы"

### Настройка после установки:

1. **Создайте группы потоков**:
   - Перейдите в "Сервисы" → "Группировка потоков"
   - Добавьте группы с названиями и ID потоков

2. **Проверьте работу**:
   - Откройте страницу "Потоки задач" (`/tasks/flow/`)
   - Убедитесь, что потоки сгруппированы по созданным разделам

## Анализ рисков при установке на продакшен

### Низкие риски:
1. **Не затрагивает ядро Bitrix** - модуль работает через стандартные API и хуки
2. **Изолированная база данных** - использует собственную таблицу `b_custom_flow_groups`
3. **Безопасное удаление** - при удалении модуля таблица удаляется, все изменения обратимы

### Средние риски:
1. **Зависимость от структуры DOM** - JavaScript ищет элементы по CSS-классам Bitrix24
   - **Решение**: Протестировать на продакшене в нерабочее время
   - **Фолбэк**: Если скрипт не сработает, таблица отобразится стандартно через 5 секунд

2. **Пути к файлам** - используются абсолютные пути через `$_SERVER['DOCUMENT_ROOT']`
   - **Решение**: Проверить корректность путей на продакшене

3. **Ссылки на пользователей** - в коде есть жестко закодированный путь `/company/personal/user/`
   - **Решение**: Убедиться, что на продакшене используется такая же структура URL

### Потенциальные проблемы:
1. **Конфликты с другими модулями** - если другие модули тоже модифицируют страницу потоков
2. **Производительность** - дополнительный JavaScript (544 строки) может замедлить загрузку
3. **Адаптивность** - CSS может конфликтовать с кастомными темами Bitrix24

## Устранение неполадок

### Если группы не отображаются:
1. Проверьте, что модуль установлен и активен
2. Убедитесь, что созданы группы с ID потоков
3. Проверьте консоль браузера на наличие ошибок JavaScript
4. Убедитесь, что таблица `b_custom_flow_groups` создана

### Если верстка "съехала":
1. Проверьте конфликты CSS с другими модулями
2. Убедитесь, что файл `flowgroups.css` загружается
3. Проверьте актуальность CSS-селекторов для вашей версии Bitrix24

### Если не работают ссылки на пользователей:
1. Проверьте путь `/company/personal/user/` на вашем портале
2. При необходимости обновите путь в `admin/flow_groups.php:210`
